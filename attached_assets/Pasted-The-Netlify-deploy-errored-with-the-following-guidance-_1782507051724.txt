The Netlify deploy errored, with the following guidance provided:

Diagnosis  
- [line 56](#L56) shows `npx next build` downloading `next@16.2.9` on the fly, which means the project does not have `next` listed in its dependencies.  
- Because the package is missing from `node_modules`, Turbopack tries to infer a workspace root outside the project and then fails to locate `next/package.json`, producing the fatal error in [lines 62-67](#L62-L67).  

Solution  
1. Open the project’s `package.json` and add the correct `next` version to the `dependencies` section (e.g., `npm install next@<your-version>`).  
2. Commit the changes to `package.json` and the updated lock file so the dependency is available during the Netlify build.  
3. Redeploy; the build should now use the locally declared Next.js package and complete successfully.
