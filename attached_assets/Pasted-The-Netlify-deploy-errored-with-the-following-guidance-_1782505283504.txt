The Netlify deploy errored, with the following guidance provided:

- **Diagnosis:** The build fails because the `next` binary is missing when the command `npm run build` executes. At [#L63-L67](#L63-L67), `npm` tries to run `next build`, but `sh: 1: next: not found` shows that the Next.js CLI isn’t installed in the production environment. This happens when `next` (and usually `react` / `react-dom`) live under `devDependencies`; Netlify installs only production dependencies (`npm install --omit=dev`), so the CLI isn’t available.

- **Solution:**  
  1. Open `package.json` and move `next`, `react`, and `react-dom` from `"devDependencies"` to `"dependencies"` (or add them there if missing). For example:
     ```json
     {
       "dependencies": {
         "next": "15.5.19",
         "react": "...",
         "react-dom": "...",
         "...": "..."
       }
     }
     ```
  2. Run `npm install` locally so `package-lock.json` reflects the change.  
  3. Commit and push the updated `package.json` and `package-lock.json`, then trigger a new Netlify deploy.

This ensures the Next.js CLI is available during the Netlify build, allowing `npm run build` to succeed.
